# Contributing

Obrigado por contribuir! Alguns pontos rápidos:

- Formato de commits: mensagens curtas e descritivas. Use branch por feature.
- Antes de abrir PR: rode `pytest`, `black` e `flake8` localmente quando possível.
- Escreva testes para mudanças que adicionam comportamento.
- Não comite credenciais (ex.: `sa-prefect.json`) — use variáveis de ambiente.

Se quiser ajuda para configurar `pre-commit`, posso adicionar a configuração.
